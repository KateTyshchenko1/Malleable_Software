🔴 Apps Were Built for Yesterday's Users

Rigid flows

Great for one prescribed path; weak the moment needs drift. Tiny tweaks require a release.

Context scattered

One task spans Notes/Maps/Calendar/Email. Facts get retyped, links get lost, nothing lines up.

Personalization is hard

Preferences are buried or global. Most people stay on defaults even when their needs differ.

Feature bloat

“Just-in-case” options pile up. The main path gets noisy; simple asks wait on backlogs.

🎯 What is Gen-UI?

Gen-UI: Software that shapes itself around what you're doing right now.

Not chatbots. Real interfaces—buttons, views, workflows—generated instantly from context and intent.

Key differences vs traditional apps

🌱 Gradual enrichment

Start freeform. Propose structure only when it clearly helps.

Begin with a note → “Add dates to calendar?” chip → accept.

👤 User agency > hard coding

The system proposes; the user confirms, adjusts, or undoes.

🎨 Design Principles for Malleable Software

  • 📝

    Start as a note, enrich gradually

    Begin unstructured; suggest fields/rules/views only when they clearly help.

  • 🔄

    One place, many views

    Same facts power list/map/calendar side-by-side—no retyping.

  • 🤫

    Keep screens quiet

    Reveal advanced controls when the user leans in.

  • 📍

    Inline, where-you-are suggestions

    Prompts next to the cursor/selection, not a separate settings view.

  • 🧩

    Small, composable blocks

    List/map/calendar/counter; avoids mega-widgets.

  • @

    Everything is referenceable

    People, places, dates, lists can be @-mentioned anywhere.
